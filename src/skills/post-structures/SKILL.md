# Post structures

Choose the best structural pattern for the post so the content flows
clearly and feels native to LinkedIn.

## When to use

- After audience and content goal are known.
- When the idea is good but the post feels shapeless.
- When the system needs a drafting scaffold.

## Inputs

topic / audience / content_goal / attribuly_pov / desired_tone

## Outputs

selected_structure / reason_for_selection / section_scaffold

## Workflow

1. Identify the post's main communication job.
2. Select the simplest structure that fits.
3. Build a section-by-section scaffold.
4. Keep the structure linear and readable.
5. Avoid overengineering short posts.

## Guardrails

- Do not use list structures when the topic doesn't benefit from them.
- Do not stack multiple structures unless necessary.
- Do not make the structure feel too formulaic.

## Quality bar

- The post should feel easy to read.
- Each section should serve a clear role.
- The structure should support the selected goal.

## Structure library

### pain_reframe_solution_question
1. Hook with pain or tension
2. Explain what people usually assume
3. Reframe the problem
4. Light solution or Attribuly angle
5. Question close

### observation_why_it_happens_pov_close
1. Operator observation
2. Why it happens
3. Why it matters
4. POV close

### contrarian_support_brand_angle_debate
1. Contrarian opener
2. Support with logic
3. Tie to Attribuly worldview
4. Invite discussion

### common_belief_wrong_better_model_cta
1. State common belief
2. Explain why it is incomplete
3. Offer better mental model
4. Soft CTA or question

### mini_case_meaning_close
1. What looked normal
2. What was actually broken
3. What changed once visibility improved
4. What this means
5. Close

### short_list_post
1. Hook
2. 3–5 points
3. Tight takeaway
4. Close
